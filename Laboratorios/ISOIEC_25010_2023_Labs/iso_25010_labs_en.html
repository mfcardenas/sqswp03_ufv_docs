ISO/IEC 25010:2023 defines a comprehensive quality model for software products, including eight key quality characteristics that software systems should possess.

This laboratory series contains 10 practical laboratories covering all quality characteristics defined in the standard, with hands-on Python programming exercises.

Laboratory List

Lab 1Introduction to Software Quality

Objective

Explore basic concepts of software quality according to the ISO/IEC 25010:2023 standard and implement functions that evaluate key quality aspects.

Problems

  • Problem 1 - Functionality Evaluation: Create a function that evaluates if an implementation meets requirements and returns a compliance percentage
  • Problem 2 - Console Usability Improvement: Design a console interface for a simple system (basic calculator) and improve usability based on ISO/IEC 25010 principles

Additional Challenges

  • Add logging to track requirements compliance
  • Implement input validations to improve robustness
  • Document how changes improve quality according to the standard

Technologies

Python 3.x with focus on functionality and usability principles. Level: Basic, for beginner students in software engineering.

Expected Output

Implemented solutions in problem1.py and problem2.py with working functionality evaluation and improved console interface demonstrating quality principles.

Lab 2Functional Suitability

Objective

Develop a deeper understanding of functional suitability by implementing systems that demonstrate functional completeness, correctness, and appropriateness.

Focus Areas

  • Functional Completeness: Ensure all specified functions are implemented
  • Functional Correctness: Verify that functions produce correct results with required precision
  • Functional Appropriateness: Ensure functions facilitate task completion appropriately

Practical Applications

  • Implement a complete task management system
  • Create comprehensive test suites to validate correctness
  • Design user-centric function interfaces
  • Evaluate compliance with functional requirements

Expected Output

A fully functional system demonstrating all three sub-characteristics of functional suitability with comprehensive testing and documentation.

Lab 3Performance Efficiency

Objective

Master performance optimization techniques and implement efficient algorithms that demonstrate time behavior, resource utilization, and capacity optimization.

Key Concepts

  • Time Behavior: Response times, processing times, throughput rates
  • Resource Utilization: CPU, memory, disk, network usage optimization
  • Capacity: Maximum limits and scalability characteristics

Practical Exercises

  • Implement and optimize search and sort algorithms
  • Profile code to identify performance bottlenecks
  • Optimize memory usage and resource consumption
  • Conduct load testing and capacity planning
  • Implement caching strategies and lazy loading

Tools and Techniques

Python profiling tools (cProfile, memory_profiler), Big O notation analysis, benchmarking frameworks, performance monitoring dashboards.

Expected Output

Optimized implementations with documented performance improvements, profiling reports, and capacity analysis demonstrating efficiency gains.

Lab 4Compatibility

Objective

Implement systems that demonstrate compatibility through co-existence and interoperability with other systems and components.

Compatibility Aspects

  • Co-existence: Ability to function efficiently while sharing environment and resources with other products
  • Interoperability: Capability to exchange information and use exchanged information with other systems

Implementation Tasks

  • Design and implement RESTful APIs for system integration
  • Create data exchange formats (JSON, XML) with schema validation
  • Implement adapters and connectors for third-party systems
  • Develop version compatibility mechanisms
  • Test multi-system integration scenarios

Technologies

REST API frameworks, JSON/XML processing, API documentation tools (Swagger/OpenAPI), integration testing frameworks.

Expected Output

Working integrations demonstrating co-existence and interoperability, comprehensive API documentation, and integration test suites.

Lab 5Usability

Objective

Create user-friendly interfaces that demonstrate usability principles including appropriateness recognizability, learnability, operability, user error protection, user interface aesthetics, and accessibility.

Usability Sub-characteristics

  • Appropriateness Recognizability: Users can recognize whether the product is appropriate for their needs
  • Learnability: Product can be learned to use easily
  • Operability: Product is easy to operate and control
  • User Error Protection: System protects users from making errors
  • User Interface Aesthetics: Pleasing and satisfying user interface
  • Accessibility: Usable by people with widest range of characteristics and capabilities

Practical Exercises

  • Design intuitive user interfaces with clear visual hierarchy
  • Implement input validation and error prevention mechanisms
  • Create helpful user documentation and tooltips
  • Develop accessible interfaces (WCAG compliance)
  • Conduct usability testing and iterate based on feedback

Expected Output

User-friendly interface demonstrating all usability sub-characteristics with accessibility features, user testing results, and improvement documentation.

Lab 6Reliability

Objective

Implement robust systems that demonstrate reliability through maturity, availability, fault tolerance, and recoverability.

Reliability Sub-characteristics

  • Maturity: System meets reliability needs under normal operation
  • Availability: System is operational and accessible when required
  • Fault Tolerance: System operates despite hardware or software faults
  • Recoverability: System can recover data and re-establish desired state after interruption or failure

Implementation Techniques

  • Implement comprehensive error handling and exception management
  • Create retry mechanisms and circuit breakers
  • Develop backup and recovery procedures
  • Implement health checks and monitoring systems
  • Design failover mechanisms and redundancy
  • Create disaster recovery plans and test them

Testing Approaches

Fault injection testing, chaos engineering, availability monitoring, recovery time testing, stress testing under failure conditions.

Expected Output

Robust system with demonstrated fault tolerance, comprehensive error handling, backup/recovery mechanisms, and reliability test results.

Lab 7Security

Objective

Implement secure systems demonstrating confidentiality, integrity, non-repudiation, accountability, and authenticity.

Security Sub-characteristics

  • Confidentiality: Data accessible only to those authorized
  • Integrity: System prevents unauthorized access to or modification of data
  • Non-repudiation: Actions or events can be proven to have taken place
  • Accountability: Actions of an entity can be traced uniquely to that entity
  • Authenticity: Identity of subject or resource can be proved to be the one claimed

Security Implementation

  • Implement authentication mechanisms (OAuth, JWT, multi-factor)
  • Develop authorization and role-based access control (RBAC)
  • Apply encryption for data at rest and in transit
  • Implement audit logging and activity tracking
  • Create secure password policies and management
  • Implement input sanitization and SQL injection prevention
  • Conduct security testing and vulnerability scanning

Security Best Practices

OWASP Top 10 mitigation, principle of least privilege, defense in depth, secure by default configuration.

Expected Output

Secure system with authentication, authorization, encryption, audit logging, and documented security testing results demonstrating all security sub-characteristics.

Lab 8Maintainability

Objective

Develop maintainable code demonstrating modularity, reusability, analyzability, modifiability, and testability.

Maintainability Sub-characteristics

  • Modularity: System composed of discrete components such that change to one has minimal impact on others
  • Reusability: Asset can be used in more than one system or in building other assets
  • Analyzability: Effectiveness and efficiency of assessing impact of intended change, diagnosing deficiencies or causes of failures
  • Modifiability: Product can be effectively and efficiently modified without introducing defects or degrading quality
  • Testability: Test criteria can be established and tests performed to determine whether criteria have been met

Best Practices

  • Apply SOLID principles for object-oriented design
  • Implement design patterns for common problems
  • Create comprehensive unit and integration tests
  • Write clean, well-documented code with meaningful names
  • Use version control effectively with meaningful commits
  • Implement continuous integration and automated testing
  • Refactor code to improve structure and reduce complexity

Code Quality Tools

Linters (pylint, flake8), code coverage tools, static analysis tools, complexity metrics (cyclomatic complexity), documentation generators.

Expected Output

Well-structured, modular codebase with high test coverage, comprehensive documentation, low complexity metrics, and demonstrated ease of modification.

Lab 9Portability

Objective

Create portable systems demonstrating adaptability, installability, and replaceability across different environments and platforms.

Portability Sub-characteristics

  • Adaptability: Product can be effectively and efficiently adapted for different or evolving hardware, software, or other operational environments
  • Installability: Product can be successfully installed and/or uninstalled in a specified environment
  • Replaceability: Product can replace another specified software product for the same purpose in the same environment

Implementation Strategies

  • Use cross-platform frameworks and libraries
  • Implement configuration management for different environments
  • Create installation scripts and deployment automation
  • Use containerization (Docker) for environment consistency
  • Implement database abstraction layers for portability
  • Create migration and rollback procedures
  • Test on multiple operating systems and platforms

Deployment Technologies

Docker, virtual environments, configuration management tools, CI/CD pipelines, package managers, infrastructure as code.

Expected Output

Portable application running on multiple platforms, automated installation procedures, environment-specific configurations, and migration documentation.

Lab 10Integrated Quality Assessment

Objective

Synthesize all ISO/IEC 25010:2023 quality characteristics into a comprehensive quality assessment framework and apply it to a complete software project.

Comprehensive Quality Assessment

  • Quality Model Implementation: Create assessment tools for all 8 quality characteristics
  • Metrics Collection: Implement automated metrics gathering for each sub-characteristic
  • Quality Scoring: Develop weighted scoring system for overall quality evaluation
  • Dashboard Development: Create visual dashboards for quality metrics
  • Report Generation: Automated quality assessment reports

Quality Characteristics Integration

  • Functional Suitability: Completeness, correctness, appropriateness metrics
  • Performance Efficiency: Time behavior, resource utilization, capacity measurements
  • Compatibility: Co-existence and interoperability assessments
  • Usability: All six sub-characteristic evaluations
  • Reliability: Maturity, availability, fault tolerance, recoverability tests
  • Security: Confidentiality, integrity, authentication, authorization checks
  • Maintainability: Modularity, reusability, testability, modifiability metrics
  • Portability: Adaptability, installability, replaceability verification

Final Project Requirements

Develop a complete software application that demonstrates all quality characteristics, implement comprehensive quality assessment tools, create quality metrics dashboard, generate quality reports, and document quality improvements throughout the development lifecycle.

Assessment Criteria

  • All 8 quality characteristics demonstrated in the application
  • Automated quality metrics collection and analysis
  • Interactive quality dashboard with visualizations
  • Comprehensive quality reports with actionable insights
  • Documentation of quality improvement process
  • Test coverage >90% with quality-focused tests

Expected Output

Complete software application demonstrating all ISO/IEC 25010:2023 quality characteristics, comprehensive quality assessment framework, automated metrics collection system, interactive quality dashboard, detailed quality reports, and complete documentation demonstrating mastery of software quality principles.
