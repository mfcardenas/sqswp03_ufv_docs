ISO/IEC/IEEE 29148:2011 is an international standard that provides comprehensive guidelines for requirements engineering processes and products in software and systems engineering.

This laboratory series contains 15 practical laboratories covering the complete requirements engineering lifecycle, from initial elicitation through maintenance and retirement.

Laboratory List

Lab 1Introduction to Requirements Engineering

Objective

Develop a comprehensive introduction to Requirements Engineering according to ISO/IEC/IEEE 29148 standard and create an interactive system that demonstrates fundamental concepts.

Requirements

  • Define requirements engineering concepts and terminology
  • Explain the requirements engineering process according to ISO/IEC/IEEE 29148
  • Create interactive examples of requirements artifacts
  • Develop a requirements classification system
  • Build a stakeholder analysis tool

Expected Output

Interactive requirements engineering introduction, requirements artifact templates and examples, stakeholder analysis tools, and requirements quality assessment framework.

Lab 2Requirements Elicitation Techniques

Objective

Master different requirements elicitation techniques and create a comprehensive elicitation management system.

Key Tasks

  • Interview Management System: Interview planning, scheduling, question generation, response recording
  • Questionnaire Builder: Dynamic questionnaire creation, multiple question types, response analysis
  • Observation Tools: Activity logging, workflow recording, time-motion studies
  • Document Analysis System: Document parsing, requirements extraction, gap analysis

Expected Output

Complete elicitation dashboard with all tools functional, Python analysis system for processing elicitation data, and documentation of a real-world elicitation process application.

Scenario

  • Dependency Identification: Find relationships between requirements
  • Prioritization: Multi-factor requirement prioritization
  • Model Generation: Functional hierarchy, data flow, state machine, use case models

Tasks

Fix intentional bugs in provided Python analysis tool and modeling dashboard, implement complete analysis functionality, create sample data and comprehensive tests.

Expected Output

Fixed Python analysis tool with all bugs resolved, working modeling dashboard with interactive features, sample requirements data, comprehensive test suite, and documentation.

Lab 14Requirements Prioritization and Negotiation

Objective

Implement advanced prioritization techniques and stakeholder negotiation support for requirements decision-making.

Prioritization Methods

  • MoSCoW: Must have, Should have, Could have, Won't have classification
  • Kano Model: Basic, performance, excitement factors analysis
  • Cost-Value: Cost-benefit and ROI-based prioritization
  • AHP: Analytical Hierarchy Process for complex decisions
  • 100-Point Method: Distributed voting for stakeholder consensus

Negotiation Support

Stakeholder voting systems, conflict resolution tools, trade-off analysis, consensus building algorithms, negotiation history tracking.

Expected Output

Complete prioritization framework with multiple methods, negotiation support tools, consensus building system, and decision documentation.
